Subnetting Without Tears: How CIDR Finally Clicked

I avoided subnetting for two weeks. Here's the mental model that made /26 and /27 stop being scary — and the 4-step method I now use on every practice question.

For two weeks I treated subnetting like a locked door. Every practice question with a slash in it — 192.168.1.0/26 — sent me straight to the next one. The breakthrough wasn't memorizing tables. It was one mental model plus a method I could run the same way every time.

#Why it kept not clicking

I was trying to do binary in my head under time pressure. That works for exactly nobody. The fix was to stop thinking in bits and start thinking in block sizes — the number of addresses each subnet owns. Once the block size is in hand, every other number falls out of it.

#The mental model that worked

A prefix like /26 means 26 network bits, so 6 host bits, so 2⁶ = 64 addresses per subnet. The block size is 64. Subnets start at multiples of the block size: 0, 64, 128, 192. That's the whole trick — find the block size, then count in jumps.

#The 4-step method

  1. Find the block size: 256 − the interesting octet value.
  2. List the subnets in jumps of the block size.
  3. The broadcast is one below the next subnet.
  4. Usable hosts are everything between network and broadcast.
subnet-plan · /26 
192.168.1.0/26   → hosts .1–.62,   broadcast .63
192.168.1.64/26  → hosts .65–.126, broadcast .127
192.168.1.128/26 → hosts .129–.190, broadcast .191

#What I'd tell past me

Stop doing binary. Find the block size, count in jumps, and the broadcast and host range fall out for free. Twenty practice questions later it's automatic — and the same method scales straight into the Network+ material.
